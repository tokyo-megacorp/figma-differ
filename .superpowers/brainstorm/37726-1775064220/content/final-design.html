Fig-Diff Dashboard v1 — Final Design

Review all decisions before writing the spec

Mission

"What changed since the last time I looked?"

Designs change while you're implementing. This tool surfaces exactly what changed between any two snapshots so you don't build against stale specs.

Architecture

review.json
+
index.json
→
render-review.js
→
review.html
Node.js generator • Calls Figma images API for S3 URLs • Embeds data + URLs as JSON • Single self-contained HTML file • Zero runtime deps

Screens

Screen 1
Index
List of available diff ranges with severity counts. Click to open a diff.
Screen 2
Accordion (triage)
All changed frames grouped by page. Expand to see diff hunks inline. Severity filter pills. GitHub PR style.
Screen 3
Detail (inspect)
Sidebar tree + full diff panel. Change groups: added, removed, bbox, visibility, components, text, fills. Lazy-loaded PNGs.

Key Decisions

Stack: Static HTML + vanilla JS, generated by Node.js
Data: Embedded JSON (review.json + index.json + image URLs)
Images: Pre-signed S3 URLs, lazy-loaded via IntersectionObserver
Triage: None — pure viewer, read-only
Theme: Dark, GitHub-ish aesthetic
UX model: GitFork-inspired, two zoom levels
Output: Single .html file, shareable via Slack
CLI: fig-diff review → generates + opens review.html

What's NOT in v1

✕ Triage / approval workflow
✕ Timeline view (v2 monitoring)
✕ Side-by-side before/after PNGs (v1.1)
✕ Build tooling / framework / dev server
✕ Persistence / localStorage
